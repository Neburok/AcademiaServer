# Conceptos Fundamentales

Este documento describe los conceptos que guían el diseño de **AcademiaServer** y del asistente **Mitzlia**.

---

# Inbox Académico

El inbox académico es la entrada principal del sistema.

Su propósito es permitir capturar ideas de forma rápida y sin fricción.

Ejemplos:

* ideas de investigación
* recordatorios
* apuntes conceptuales
* notas de lectura

El inbox funciona como el punto inicial del conocimiento.

---

# Pipeline de Procesamiento

El pipeline transforma información cruda en conocimiento estructurado.

## Etapas típicas:

- captura
- clasificación
- extracción de metadatos
- estructuración

Este enfoque permite procesar información de forma sistemática.

---

# Base de Conocimiento

La base de conocimiento es el repositorio de todas las notas procesadas.

Características:

* persistente
* estructurada
* versionable
* portable

Cada nota es una unidad de conocimiento.

---

# Eventos

Un evento es una acción que ocurre en un momento específico.

Ejemplos:

* recordatorios
* alertas
* notificaciones

El scheduler permite detectar eventos y ejecutar acciones.

---

# Infraestructura Cognitiva

AcademiaServer está diseñado como una **infraestructura cognitiva personal**.

Esto significa que el sistema busca amplificar la capacidad del usuario para:

- pensar
- organizar
- recordar
- conectar ideas

No es solo un sistema de almacenamiento.

Es una extensión digital del pensamiento.

---

# Mitzlia

Mitzlia es el asistente que utiliza esta infraestructura para interactuar con el usuario.

Sus funciones incluyen:

- capturar conocimiento
- organizar información
- recordar eventos
- asistir procesos académicos

Mitzlia representa la interfaz cognitiva del sistema.

---

# Filosofía del Proyecto

El proyecto sigue una regla simple:

- Primero infraestructura
- Luego automatización
- Después inteligencia
